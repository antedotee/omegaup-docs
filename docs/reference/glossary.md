# Glossary

Common terms and definitions used throughout omegaUp documentation.

## General Terms

**omegaUp**  
The educational programming platform that helps students improve their programming skills.

**Contest**  
A timed programming competition with multiple problems.

**Problem**  
A programming challenge with test cases and evaluation criteria.

**Submission (Run)**  
Code submitted by a user to solve a problem.

**Arena**  
The contest interface where participants solve problems.

## Technical Terms

**Grader**  
The service that manages submission queues and coordinates evaluation.

**Runner**  
A service that compiles and executes user-submitted code in a secure sandbox.

**Minijail**  
Linux sandbox used for secure code execution (forked from Chrome OS).

**DAO (Data Access Object)**  
PHP classes that handle database interactions.

**VO (Value Object)**  
PHP classes that map to database tables.

**MVC (Model-View-Controller)**  
Architectural pattern used in omegaUp.

## Verdicts

**AC (Accepted)**  
Solution is correct.

**WA (Wrong Answer)**  
Solution produces incorrect output.

**TLE (Time Limit Exceeded)**  
Solution exceeded time limit.

**MLE (Memory Limit Exceeded)**  
Solution exceeded memory limit.

**RTE (Runtime Error)**  
Solution crashed during execution.

**CE (Compilation Error)**  
Code failed to compile.

**PA (Partially Accepted)**  
Solution passed some but not all test cases.
